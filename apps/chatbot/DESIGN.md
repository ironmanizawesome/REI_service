# 이음 챗봇 설계 메모

논의 기반 설계 결정 정리 (규현). 구현 전 합의 사항 기록.

## 1. 아키텍처 — 3층 (정확성 우선, LLM 환각 차단)

| 층 | 처리 방식 | 근거 |
|----|-----------|------|
| **REI 수치** | `data/rei_results.json` 직접 룩업 (LLM 아님) | 재출입 시간은 지어내면 안 됨. 계산·룩업. |
| **방제 사이클 추천** | 결정론적 규칙 엔진. LLM은 "왜"만 설명 | 추천 정확성 = 저항성/안전 직결 |
| **정보성 Q&A** | 진짜 RAG (검색+생성) | "왜 이 기간?", 방법론, 라벨 주의 등 |

## 2. 모델

- **1차: OpenAI 통일** (임베딩+생성). 마감(7/10) 고려, 세팅 최속.
- 검색 품질 아쉬우면 **임베딩만 bge-m3(한국어) 등으로 교체**. 교체 비용 = 함수 한 줄 + 인덱스 재생성.
- 임베딩 호출은 `rag.get_embeddings()` 한 곳에 격리해 교체 쉽게.

## 3. REI 룩업 — 되묻기 필요

룩업 키 = (성분 × 작물 × **작업유형** × **작업시간**). 4차원.
- 작업유형: 수확 / 예찰
- 작업시간: 1·2·4·6·8h — REI는 성분보다 작업시간에 민감(데이터 확인됨)
- → 챗봇은 "무슨 작업? 몇 시간?"을 되물어야 정확한 값 제공
- 예찰은 1·2h만 산정(전부 0). 수확이 실질 대상.
- 출처: EFSA 2022 Guidance (`rawdata/`)

## 4. 방제 사이클 추천 엔진 — IRAC는 한 축일 뿐

교수님 지목 "진짜 가치 있는 기능" = "오늘 이거 쳤으니 다음엔 뭐 쳐라". 다축 규칙:

1. **대상 해충/병** — 지금 뭘 잡나 + 시즌상 다음 위협. (응애 사이클 ≠ 진딧물 사이클)
2. **살포 주기·시기** — 예방=달력 기반 주기 살포. 해충 생활사·생육단계 기반. (IRAC 무관 축)
3. **저항성 로테이션** — 살충제 IRAC / 살균제 FRAC. 같은 그룹 연속 회피.
4. **PHI + 라벨 최대 사용횟수** — 수확 임박 제약, 시즌당 횟수 제한.
5. **희석 배율·사용량(mL)** — 배액이 노출 크게 좌우(교수님). 어르신 대상 정확 mL 안내(김기현 대표).
6. **실제 농가 사용 이력** — 김기현 대표 수년치 실제 농약명이 출발점(교수님). 추상 스킴 아님.

→ 엔진 흐름: 현재 성분 입력 → 대상 해충 파악 → (같은 해충 & 다른 IRAC 그룹 & PHI/횟수 OK) 후보 → 주기 반영 추천 → LLM이 이유 설명.

## 5. 성분 6종 + IRAC 그룹 (⚠️ 미검증 초안)

전부 살충·살비제. IRAC 원본(MoA Classification) 대조 필요.

| 성분 | 분류 | 대상(추정) | IRAC |
|------|------|-----------|------|
| Hexythiazox | 살비 | 응애 | 10A |
| Fenpyroximate | 살비 | 응애 | 21A |
| Spirodiclofen | 살비 | 응애 | 23 |
| Bifenazate | 살비 | 응애 | 20D |
| Acetamiprid | 살충 | 진딧물/가루이 | 4A |
| Chlorantraniliprole | 살충 | 나방(나비목) | 28 |

→ 로테이션 실익은 응애약 4종 사이(IRAC 다 다름).

## 6. 미뤄둔 작업

- **살균제(FRAC) 경로** — 살려두되 **맨 마지막 작업**. 구조 자리만 확보(class="fungicide" + frac_group). 그때 포함 여부 판단.

## 7. 열린 액션 아이템

- [ ] **김기현 대표 실제 사용 농약 리스트** 확보 — 사이클 엔진 핵심 입력. 없으면 6종 축소판 선진행.
- [ ] 성분별 **대상 해충 데이터** 확정 (팀 자료 or PSIS 라벨 초안)
- [ ] **IRAC 그룹 검증** (채윤 작용기작 분류 자료와 대조)
- [ ] **EPA REI 원본 확인** — 섭취 아닌 접촉·작업 기준 확인됨. **희석 전/후 어느 상태 산정인지** 원본 문헌 직접 확인(GPT/클로드 말고).
- [ ] 희석 배율(배액) REI 반영 — 현재 원액 살포처럼 계산됨
- [ ] 온실 수경재배 조건 반영 (미국 노지 EPA 값 부적절 지적)

## 8. 발전 방향 (교수·대표 제안)

- **반감기 감소 곡선 시각화** — 위험→주의→안전 구간. "언제부터 안전" 단칼보다, 곡선 보여주고 농가가 스스로 판단. (엄격 구간 기준은 피지컬 AI 논거로도 세분화 여지)
- **정확한 사용량(mL) 안내** — 작물 입력 시. 라벨 못 읽는 어르신 대상.
- 규제화보다 **자발적 활용** 프레이밍(피크 시간 최소 작업, 갓난아기 농가 등).
